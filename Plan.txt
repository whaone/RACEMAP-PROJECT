# Frontend PRD — Racemap Live Tracking Platform

## 1. Product Overview

Racemap is a real-time live tracking platform for cycling race events.

The platform uses:

* Strava Beacon links as participant live tracking sources.
* OpenStreetMap as the primary map provider.
* Checkpoint-based progression tracking.
* Real-time leaderboard and participant monitoring.

The frontend acts as:

* Public live race viewer.
* Race control dashboard.
* Real-time monitoring interface.
* Spectator engagement platform.

This document is specifically designed for AI developer agents to understand and implement the frontend architecture, UI system, responsive behavior, and real-time interactions.

---

# 2. Product Goals

## 2.1 Primary Goals

The frontend must:

1. Display participant positions in real-time.
2. Display race rankings.
3. Display checkpoint progression.
4. Display the full event route.
5. Support multi-participant comparison.
6. Work smoothly on desktop and mobile devices.
7. Be easy to understand for spectators.
8. Remain performant even with many participants.

---

# 3. Core Business Rules

## 3.1 Ranking Logic

Primary ranking rules:

1. Participants who finish first rank higher.
2. If participants have not finished yet:

   * Latest completed checkpoint.
   * Distance toward next checkpoint.
   * Checkpoint timestamp.

Important:

* The frontend only consumes ranking data from the backend.
* The frontend must not calculate heavy race logic.

---

# 11. Real-Time Behavior

## Live Updates

Frontend must:

* Update markers smoothly.
* Animate movement.
* Avoid hard rerenders.
* Preserve map zoom state.

---

## Suggested Update Interval

* Every 2 to 5 seconds.

---

## Stale Data Handling

If a participant stops updating:

* Show warning badge.
* Reduce marker opacity.
* Update leaderboard status.
